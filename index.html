Automated System Monitoring Script

Description: This project implements a Bash script designed to monitor critical system resources on Linux servers, including CPU usage, memory utilization, and disk space. It automatically sends email alerts to administrators when resource thresholds are exceeded, enabling proactive identification and resolution of potential performance issues. Demonstrates skills in Bash scripting, system administration, and proactive system monitoring.


#!/bin/bash
# System Monitoring Script

CPU_THRESHOLD=80
MEM_THRESHOLD=90
DISK_THRESHOLD=95

cpu_usage=$(top -bn1 | grep "Cpu(s)" | sed "s/.*, *\([0-9.]*\)%* id.*/\1/" | awk '{print 100 - $1}')
mem_usage=$(free | awk 'NR==2{printf "%.2f", $3/$2 * 100}')
disk_usage=$(df -h / | awk '$NF=="/"{printf "%s", $5}' | sed 's/%//g')

if (( $(echo "$cpu_usage > $CPU_THRESHOLD" | bc -l) )); then
  echo "CPU usage exceeds $CPU_THRESHOLD%: $cpu_usage%" | mail -s "CPU Alert" shahidakilgori@gmail.com
fi

if (( $(echo "$mem_usage > $MEM_THRESHOLD" | bc -l) )); then
  echo "Memory usage exceeds $MEM_THRESHOLD%: $mem_usage%" | mail -s "Memory Alert" shahidakilgori@gmail.com
fi

if (( $(echo "$disk_usage > $DISK_THRESHOLD" | bc -l) )); then
  echo "Disk usage exceeds $DISK_THRESHOLD%: $disk_usage%" | mail -s "Disk Space Alert" shahidakilgori@gmail.com
fi

int main() {
    FILE *fp;
    char str[100];

    // Open file for writing
    fp = fopen("my_file.txt", "w");
    if (fp == NULL) {
        perror("Error opening file");
        return 1;
    }
    fprintf(fp, "Hello, this is a test.\n");
    fclose(fp);

    // Open file for reading
    fp = fopen("my_file.txt", "r");
    if (fp == NULL) {
        perror("Error opening file");
        return 1;
    }
    fgets(str, 100, fp);
    printf("Read from file: %s", str);
    fclose(fp);

    return 0;
}
